## What I Build

**Frameworks and engines**

- [Score](https://github.com/bwyard/score) — EDM audio engine with a functional GUI.
  Songs are pure functions of time. Custom DSL, real-time DSP pipeline, WASM core.
- [Prime](https://github.com/bwyard/prime) — Rust math library. Foundations for the
  Score/Stage/Form ecosystem: Time, Causality, Information, Context.
- [Form](https://github.com/bwyard/form) — Procedural graphics framework. Scenes as
  pure functions of space. Math-first, SDF-based.
- [Stage](https://github.com/bwyard/stage) — Game domain systems. Combat, economy,
  quest graphs, AI. 505 tests.
- [Evalocity](https://github.com/bwyard/evalocity) — LLM evaluation framework. In progress.

**Client platforms**

Enterprise-grade SvelteKit 5 and React monorepo builds for creative, performance,
real estate, and entertainment businesses. Deployable self-hosted or platform-hosted.
Built to operate without a full-time dev staff. Active client work across creator
monetization, performer platforms, ticketing, and real estate. Client information
stays private — examples and case studies available on request.

**Enterprise tooling and libraries**

Custom MCP servers, developer tooling, and data generation libraries built to
enterprise standards. Examples include a synthetic data generation library for
military medical health data — outputs directly to code, JSON, and test fixtures,
with 2000+ tests. Tooling built for real adoption across large engineering teams,
not just internal use.

**Enterprise test infrastructure**

- Built a shared test infrastructure layer consumed by 3 independent applications —

- Designed and shipped an MCP tool adopted organically across teams without mandates —
  now a standard interface across a large multi-project enterprise program
- Implemented accessibility compliance tooling (jest-axe, cypress-axe) across 3 apps —
  defined the standard, not just ran the tests
- Built a domain-specific synthetic data generation library — fixture generation covers
  complex domain data structures without touching production data
- Architected a Testcontainers base class layer enabling consistent containerized
  test environments across CI — reusable across teams
- Identified and presented architectural refactor recommendations to DB and dev leads —
  both accepted, both ticketed and committed

**AI and MCP tooling**

MCP server development, custom session management systems, AI workflow integration,
prompt engineering. Internal tooling with organic adoption across large engineering
teams. Tools: Claude, GitHub Copilot, OpenCode.

**Privacy-first architecture**

I build platforms where sensitive data is handled correctly from the start — consent

Active client work in this space — examples available on request.
